The 7 Levels of Claude Code & RAG | Chase AI סרטון זה מפרק לעומק את אתגר הזיכרון במערכות AI, ומציג שבע רמות התפתחות החל משימוש בסיסי בקובצי .claude.md ועד למערכות Graph RAG מורכבות [1, 2]. נלמד כיצד לבנות ארכיטקטורת זיכרון יעילה באמצעות ספריות Obsidian מקומיות במקום להסתמך על חלון הקשר עמוס שמוביל לתופעת "ריקבון ההקשר" (Context Rot) וירידה בביצועים [3-5]. תובנה מרכזית שעולה היא שעבור רוב המשתמשים, ניהול זיכרון ידני ומובנה ב-Obsidian מהווה פתרון של 99% החוסך את העלויות והמורכבות הטכנית של הקמת מערכות RAG כבדות [6, 7]. בנוסף, מודגמת חשיבות החלוקה של קובצי הקשר נפרדים למשימות ספציפיות כדי לשמור על מיקוד הסוכן ולהימנע מזיהום הקשר (Context pollution) [8-10]. עבור פסיכיאטר וחוקר, הבנה זו קריטית לבניית סביבת טרמינל מקומית המסוגלת לנתח רשומות קליניות ומאמרים רפואיים ארוכים באופן מדויק ואמין, מבלי לאבד הקשר וללא תשלום מופקע על טוקנים.

I Replaced Karpathy's LLM Wiki with Something That Actually Works | Artem Zhutov הסרטון מציג השוואה מעשית בין גישת ה-"LLM Wiki" המקומית של אנדריי קרפתי לבין שימוש ב-NotebookLM לייעול תהליכי עיבוד ידע, במטרה להפוך מידע גולמי לכלים אופרטיביים בטרמינל [11-13]. הודגם כיצד כריית ידע ישירות למערכת ויקי מקומית דורשת זמן רב ועלויות טוקנים אדירות במעבר על תמלילים מלאים [14, 15], בעוד ששילוב חכם של בניית מפות חשיבה מבוססות הקשר מאפשר לזקק עקרונות פעולה במהירות [13, 16]. התובנה המרכזית היא שלא מספיק לאגור ידע; יש להמיר אותו באמצעות Claude Code ל"כישורים" (Skills) אוטומטיים הניתנים להפעלה יומיומית [13, 17, 18]. טכניקה זו של בניית "כישורים" רלוונטית במיוחד לפרופיל שתואר, שכן היא מאפשרת להזין פרוטוקולים פסיכיאטריים וממצאי מחקר עדכניים ולהפוך אותם לסוכני סיוע קליני מותאמים אישית שרצים ישירות משורת הפקודה ב-macOS.

100 Hours Testing Claude Code vs Antigravity (honest results) | Nate Herk | AI Automation מבדק מקיף המשווה בין Claude Code, הפועל ככלי CLI ממוקד טרמינל, לבין Google Antigravity, המשמש כסביבת פיתוח (IDE) עצמאית [19-21]. הסרטון מדגים את עוצמתו של מנגנון התכנון המקדים (Planning Mode) ב-Claude Code, אשר מנתח וממפה ארכיטקטורות קבצים טרם ביצוע שינויים בקוד [22, 23]. עולה כי Claude Code מצטיין בשילוב שרתי מודלים מרובים דרך פרוטוקול MCP, אותם ניתן להגדיר דרך קובצי JSON גלובליים או פר-פרויקט ישירות בטרמינל [24, 25]. כמו כן, מודגש הצורך הקריטי בניהול נכון של טוקנים, חלוקת משימות קטנות וניקוי חלון השיחה כדי למנוע אובדן הקשר לאורך זמן [26, 27]. למי שעובד ב-macOS ונסמך על הטרמינל, סרטון זה מספק אשרור מתודולוגי לבחירה ב-Claude Code כפלטפורמה המרכזית וחושף כיצד לשלוט על כלי העזר (CLI tools) באופן רציף ואורגני [21, 25].

My Claude Code workflow no one knows about | Greg Isenberg סרטון זה חושף זרימת עבודה מתקדמת מבוססת טרמינל, המשלבת שרתי MCP מרובים כיחידה אוטונומית ב-Claude Code [28-30]. הערך המרכזי הוא ההדגמה של הטרמינל כ"מערכת הפעלה" של ממש, המנהלת סוכנים שמבצעים מחקר, עורכים קוד ושואבים דאטה חיצוני באופן רוחבי [31, 32]. אף שהדוגמאות ממוקדות בשיווק, התפיסה הארכיטקטונית של שילוב זרימות עבודה מרובות סוכנים ישירות דרך ה-CLI יכולה לעצב מחדש תהליכי עיבוד נתונים ומחקר אקדמי.

How Robots Learn & Adapt in Real Life | ערוץ: IBM Technology הסרטון מציג את קפיצת המדרגה של בינה מלאכותית מעולם הדיגיטל ("ביטים") לעולם הפיזי ("אטומים") באמצעות מעבר לסוכנים המבוססים על מודלי חזון-שפה-פעולה (VLAs) [1-3]. בניגוד לתכנות רובוטי ישן מבוסס-חוקים, מודגמת גישה של שילוב הבנת עולם כללית (דרך מודלי שפה וראייה) יחד עם למידת חיזוק (Reinforcement Learning) בתוך סביבות וירטואליות [4-6]. מבחינת תובנות מרכזיות, הסרטון חושף כי המהפכה מתאפשרת בזכות יעילות עיבוד חסרת תקדים של מעבדי GPU, אשר מסוגלים לעבד 20 מיליון שעות וידאו בשבועות בודדים במקום בשנים [7]. בנוסף, מוסבר תהליך ה"רנדומיזציה של הדומיין" (Domain Randomization) — שינוי מכוון של משתנים כמו תאורה או חיכוך בסימולציה כדי לייצר למידה גמישה שתשרוד בעולם האמיתי [8, 9]. עבור פסיכיאטר ומפתח הבונה סוכנים קליניים ומשתמש ב-MCP, הסרטון חשוב במיוחד מאחר ומושג ה-sim-to-real gap (גישור בין סימולציה למציאות רועשת) [10, 11] מקביל לחלוטין לאתגר שבמעבר מסביבת הפיתוח והבדיקות בטרמינל (ה"סימולציה"), אל העולם המורכב, הבלתי-צפוי והאמוציונלי של אינטראקציות אמיתיות עם ילדים ונוער בקליניקה.
