🤖 Claude Code & Agents

🔥 Tier A — חובה

Full Workshop: Build Your Own Deep Research Agents | AI Engineer בסדנה זו מוצגת ארכיטקטורה מלאה לבניית סוכן מחקר מעמיק, תוך שימוש ב-Claude Code כמעטפת לסוכן (Agent Harness) וב-FastMCP לניהול כלי השרת [1, 2]. המערכת מפצלת את העבודה לשלב מחקר אקספלורטיבי ודינמי, ולשלב כתיבה דטרמיניסטי [3]. הסרטון מדגים כיצד הסוכן משתמש ב-API של Gemini כדי לחפש ברשת ולנתח תמלילי וידאו מ-YouTube באופן אוטומטי [4, 5]. תובנה מרכזית שעולה היא הצורך לנהל "ריקבון הקשרי" (Context Rot) על ידי האצלת סמכויות לכלים חיצוניים ולסוכני-משנה בעלי חלון הקשר נפרד [6]. כמו כן, מודגם שימוש ב-LLM כשופט (LLM-as-a-judge) במסגרת לולאת אופטימיזציה להערכת איכות הפלטים [7, 8]. עבור פסיכיאטר וחוקר, סרטון זה הוא אוצר; הוא מספק מפת דרכים מדויקת לשימוש ב-Claude Code ו-MCP כדי לאטמט סקירות ספרות קליניות ומחקריות ישירות מהטרמינל, תוך הפקת פלטים מובנים המגובים במקורות [1, 9].

How to Never Hit Your Claude Session Limit Again | Nate Herk | AI Automation הסרטון מציג אסטרטגיות קריטיות לניהול חלון ההקשר (Context Window) ב-Claude כדי לחסוך טוקנים ולשמר את אינטליגנציית המודל [10, 11]. מוסבר בפירוט התהליך של "ריקבון הקשרי" (Context Rot), שגורם לירידה בדיוק השליפה מ-92% ל-78% כאשר חלון ההקשר מתמלא והמודל מתחיל להיות מוסח ו"להזות" [12]. כדי להתמודד עם זה, מודגמות טכניקות פרקטיות כמו דחיסה ידנית של ההקשר (Compact/Clear), שימוש בסוכני-משנה למשימות ממוקדות, והמרה של מסמכי PDF ו-Word ל-Markdown — פעולה שחוסכת עד 70% מהטוקנים ב-Claude Code [13-15]. תובנה חשובה נוספת היא המשמעות של תחזוקה קפדנית של קובץ ה-claude.md, שמגדיר את הפרוייקט ונטען בכל פעם מחדש [16]. למי שעובד מהטרמינל עם Claude Code באופן יומיומי למחקר, הכלים וההרגלים המודגמים כאן הם חובה למניעת השתגעות המודל, לחיסכון במשאבים, וליעילות אופטימלית בעיבוד חומר קליני רב ומסמכים רפואיים.

Hermes Agent: The New OpenClaw? | Greg Isenberg סרטון זה צולל ליכולות של Hermes Agent כחלופה יציבה וחכמה ל-OpenClaw, עם דגש חזק על מערכת הזיכרון המובנית שלו המבוססת על מסד נתונים SQLite [17, 18]. המערכת מאפשרת לסוכן ללמוד ברציפות מתהליכי העבודה של המשתמש, לזכור פרטים, ולבצע חיפושים בהיסטוריית הפעולות המוצלחות שלו [18]. בנוסף, מודגמת בו אינטגרציה עמוקה עם תוכנת Obsidian, שבה הסוכן מארגן, מסדר ומייצר קבצי Markdown באופן אוטומטי המהווים לוח בקרה אישי [19, 20]. הסוכן מגיע עם מעל 40 כלים מובנים שימושיים (כמו Cron jobs) וניתן להרצה אפילו דרך טרמינל (Termux) במכשירי אנדרואיד [21-23]. עבור רופא העובד מול macOS ומשתמש בסיכומים ומחקר, סוכן זה מציע דרך פרקטית לנהל "מוח שני" ב-Obsidian, לאטמט משימות מחקר מחזוריות ולשמור על סביבת עבודה טרמינלית שמתפתחת ולומדת את צרכיו [20, 24].

👀 Tier B — שווה סריקה

Hermes Agent is insane… 100,000+ github stars | David Ondrej הסרטון מציג כיצד להתקין את Hermes Agent על גבי שרת מרוחק (VPS) דרך חיבור SSH בטרמינל, מה שמאפשר לסוכן לרוץ ברקע באופן רציף [25, 26]. מודגם בו שימוש בכלי "Browser Harness" שמעניק לסוכן יכולת אוטונומית לשלוט בדפדפן, לגרד מידע מאתרים, ואף לתקן את עצמו במקרה של שגיאות (Self-healing) [27-29]. הערך המרכזי כאן הוא היכולת לחבר את הסביבה הטרמינלית למשימות אוטומציה מורכבות, כמו איסוף נתונים ממאגרי מידע רפואיים או פורומים, בצורה אוטומטית ויעילה לחלוטין.

Gemma, DeepMind's Family of Open Models | AI Engineer הרצאה הסוקרת את משפחת מודלי Gemma 4 של גוגל, בדגש על ארכיטקטורת "E2B" (Effectively 2 Billion) המאפשרת הרצה מהירה שלהם על מכשירי קצה כמו טלפונים ומחשבים ניידים באמצעות CPU וזיכרון מוגבל [30, 31]. מוזכרות גם גרסאות ייעודיות כמו MedGemma שעברו כוונון למשימות קליניות ורפואיות [32]. הערך המרכזי עבורך הוא העמקת היסודות (LLM fundamentals) במודלי שפה פתוחים שניתנים להרצה מקומית, דבר שקריטי לשמירה על פרטיות מידע מטופלים בעבודה פסיכיאטרית.

The New Application Layer - Malte Ubl, CTO Vercel | AI Engineer ה-CTO של Vercel מסביר כיצד סוכני AI הופכים למשתמשים העיקריים של תשתיות תוכנה ו-API, כאשר מעל 60% מהתעבורה באתר שלהם מגיעה מסוכנים [33, 34]. הוא מציע גישה פרגמטית: בניית סוכנים פשוטים שמבצעים את "שלב המחקר" ואיסוף המידע האוטומטי, ומשאירים את קבלת ההחלטות לאדם [35]. הערך כאן הוא בניית תפיסת עולם מדויקת לשילוב סוכנים בעבודה הקלינית, המעדיפה פתרונות אוטומציה של הטרחה השוחקת במקום מערכות קבלת החלטות עצמאיות מדי [36].

Claude Design + Seedance 2.0 = INSANE Animated Websites | Chase AI מדריך בניית אתרים המדגים כיצד לשלב כלי ייצור תמונות (Nano Banana) ואנימציית וידאו (Seedance) יחד עם Claude Design ליצירת דפי נחיתה מונפשים [39, 40].

📈 מגמה מרכזית

המגמה הבולטת העולה מסרטוני השבוע היא המעבר מסוכני AI מונעי פרומפטים מבודדים, לסביבות עבודה מורכבות הרצות בטרמינל או מקומית וכוללות זיכרון מתמשך, ניהול משאבים ויכולות פריצה החוצה אל הרשת. אנו רואים זאת בארכיטקטורת המחקר המשתמשת בפרוטוקול MCP להפרדה בין "המוח" (Claude) לבין הכלים שהוא מפעיל (סריקת רשת, ניתוח וידאו) [1, 2]. דוגמה נוספת היא Hermes Agent, שרותם מסד נתונים מקומי מסוג SQLite לניהול היסטוריה ארוכת טווח ומשתמש ב-Browser Harness לפעולות רשת אוטונומיות [17, 27]. בנוסף, מתפתחת דיסציפלינה שלמה של ניהול חלון ההקשר (Context Management) והמרה לפורמטים רזים (כמו Markdown), מתוך הבנה שכדי שהסוכנים יהיו אמינים לאורך סשנים מחקריים ארוכים, חובה למנוע מהם לקרוס לתוך "ריקבון הקשרי" [12, 15].

What AI Agent Skills Are and How They Work | ערוץ: IBM Technology סרטון זה מסביר כיצד "מיומנויות" (Skills) מעניקות לסוכני AI ידע פרוצדורלי – היכולת לבצע משימות רב-שלביות ברצף, בניגוד לידע עובדתי גרידא שהיה שמור להם עד כה [1, 2]. המיומנות מוגדרת באופן אלגנטי ופשוט כקובץ skill.md הכולל מטא-דאטה (YAML front matter עם שם ותיאור) והנחיות בפורמט Markdown, ולעיתים גם סקריפטים (JavaScript/Python) ומשאבים נלווים בתיקיות אופציונליות [3, 4]. הסרטון חושף מנגנון אופטימיזציה בשם "חשיפה הדרגתית" (Progressive Disclosure) החוסך בעלויות ובאסימונים (tokens): הסוכן טוען רק את התיאורים לזיכרון הראשוני, ושולף את ההוראות והסקריפטים המלאים רק כאשר המודל מזהה התאמה למשימה [5-8]. שתי תובנות קריטיות שעולות הן ההבחנה החדה בין פרוטוקול MCP (שמספק יכולת טכנית להפעיל כלי) לבין מיומנות (שמספקת את שיקול הדעת מתי וכיצד להפעיל אותו), וההקבלה העמוקה למדעי הקוגניציה דרך חלוקת הארכיטקטורה לזיכרון סמנטי (RAG), אפיזודי (היסטוריית שיחה) ופרוצדורלי (Skills) [8-12]. הרלוונטיות לפרופיל שלך גבוהה במיוחד: מאחר שהתקן פתוח ונתמך ב-Claude Code, תוכל כמטפל וחוקר לקודד פרוטוקולים קליניים מורכבים או תהליכי ניתוח דאטה לתוך קבצי Markdown פשוטים שירוצו אצלך בטרמינל ה-macOS, אך עליך להיזהר מסיכוני אבטחה (כגון חדירת נוזקות או Prompt Injection) מאחר שמדובר בהרצת סקריפטים מקומית [10, 12, 13].

המגמה המרכזית העולה מהסרטון היא המעבר מיצירת סוכני AI שמסתמכים אך ורק על ידע עובדתי וכלים נפרדים, לסטנדרטיזציה קוד פתוח של "ידע פרוצדורלי" – מתן זיכרון תהליכי למודלים [1, 12, 14]. במקום להעמיס את הפרומפט הראשוני בעשרות שלבי עבודה, התעשייה עוברת לשימוש בתקנים גלויים, דוגמת agentskills.io, שמאפשרים ניידות של תהליכי עבודה בין פלטפורמות שונות כמו Claude Code ו-OpenAI Codex [2, 10]. הגישה הזו שואבת השראה ישירה מהפסיכולוגיה הקוגניטיבית (הפרדה בין זיכרון אפיזודי, סמנטי ופרוצדורלי), ומאפשרת לסוכנים להפעיל שיקול דעת עצמאי בבחירת פרוטוקולי העבודה הנכונים מתוך רשימת אינדקס קלה, מבלי לסתום את חלון ההקשר (Context Window) [5, 6, 11, 12].

לחלופין, אם תרצה, אני יכול לחפש עבורך את התוכן העדכני ביותר ברשת בנושאים שציינת.
