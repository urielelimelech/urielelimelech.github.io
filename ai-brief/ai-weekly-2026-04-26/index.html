Full Walkthrough: Workflow for AI Coding from Planning to Production | AI Engineer Matt Pocock מדגים תהליך עבודה מקיף לכתיבת קוד עם סוכנים, תוך דגש על ניהול "אזור הטיפשות" (Dumb Zone) שמכה במודלים כשחלון ההקשר שלהם עובר את רף ה-100K טוקנים [1, 2]. הוא מציג פרקטיקות כמו מיומנות ה-"Grill Me", הפוקדת על ה-AI לתחקר את המשתמש עד להגעה להבנה משותפת ויצירת מסמך דרישות (PRD) מדויק [3-5]. 2-3 תובנות שעולות: (1) מודלים נוטים לכתוב קוד בצורה אופקית (שכבות), אך אילוצם לפתח בשיטת "Tracer Bullets" (חיתוך ורטיקלי) מאפשר פידבק מקצה-לקצה בשלב מוקדם של הפרויקט [6-8]. (2) פיתוח מונחה-בדיקות (TDD) חיוני כדי למנוע מהסוכן "לרמות" ולהתאים בדיקות לקוד שגוי שהוא עצמו כתב [9, 10]. (3) עדיף למחוק לחלוטין את חלון ההקשר בין משימות (להיות כמו "גיבור הסרט ממנטו") מאשר לכווץ אותו, כדי להבטיח את מלוא חדות המודל [11-13]. למה זה רלוונטי: עבור פסיכיאטר הכותב סקריפטים מחקריים בטרמינל, הבנת מגבלות ה-Context חיונית לעיבוד מאגרי מידע; מתודולוגיית תחקור הסוכן תעזור לזקק רעיונות מחקריים לפרומפטים שימושיים ולמנוע כתיבת קליפורים פגומים.

MCP = Mega Context Problem | AI Engineer Matt Carey סוקר את הבעיה האקוטית בפרוטוקול MCP – טעינת ספציפיקציות API שלמות ככלים (Tools) גורמת ל"פיצוץ" חלון ההקשר (למשל, ה-API של Cloudflare צורך 1.1M טוקנים) והורסת את יכולת ההסקה של המודל [14, 15]. הפתרון המוצג הוא מעבר ל"גילוי פרוגרסיבי", בו סוכנים כותבים בעצמם קוד TypeScript דינמי כנגד ה-API ומריצים אותו בתוך סביבות מבודדות (Sandboxes) מאובטחות [16-18]. 2-3 תובנות שעולות: (1) מתן גישת CLI סטנדרטית לסוכן היא נוחה (כמו ב-Claude Code) אך יוצרת סיכון אבטחה חמור (CVE) אם אין בידוד [19, 20]. (2) קוד הופך להיות הפורמט הדחוס והיעיל ביותר להבעת תוכנית עבודה של מודל בהשוואה לקריאות JSON מוגדרות מראש [21]. (3) המגמה תהיה שממשקי API יהיו חייבים לעמוד בעומסי סוכנים שיריצו לולאות קוד אוטומטיות כנגדם [22]. למה זה רלוונטי: כמי שמעמיק ב-MCP עבור עבודתו הקלינית והמחקרית, ההבנה שאין לדחוף כלי EMR או מאגרי ספרות רפואית ל-Context במלואם, אלא לאפשר כתיבת סקריפטים בסנדבוקס, תאפשר לך לשאוב נתונים מאסיביים בבטחה מבלי לרוקן את מודל השפה מיכולותיו [14, 20].

Claude Code + Playwright Automates Literally Anything | Nate Herk | AI Automation הסרטון מציג שילוב עוצמתי בין Claude Code בטרמינל לממשק הפקודה Playwright CLI לצורך אוטומציה של פעולות גלישה מורכבות [23, 24]. הסוכן מתכנן, כותב סקריפטים ובודק אותם, תוך איסוף נתוני קשר, מילוי טפסים רפואיים/עסקיים, וניווט בפלטפורמות מאובטחות הדורשות ניהול עוגיות (Cookies) ומשתמש [25-27]. 2-3 תובנות שעולות: (1) עבודה מול דפדפן באמצעות CLI חוסכת כמות עצומה של טוקנים לעומת שימוש בשרתי MCP כבדים כמו Chrome DevTools [24, 28, 29]. (2) הסוכנים נכנסים ללולאות "ריפוי עצמי" אקטיביות – קוראים את צילומי המסך שהם לוקחים, מזהים שגיאות UI ומתקנים את הקוד בעצמם [30-32]. למה זה רלוונטי: יכולת זו הופכת את הטרמינל ב-macOS שלך למרכז פיקוד; תוכל להקים סוכנים שישאבו מאמרים מחשבונות אוניברסיטאיים סגורים, יריצו אוטומציות במערכות זימון תורים, או יבצעו QA התנהגותי לשאלונים פסיכיאטריים דיגיטליים שאתה מפתח לבני נוער [23, 32, 33].

המגמה הבולטת ביותר שעולה מהסרטונים השבוע היא זינוק בהתאמתם של סוכני AI ומודלים לסביבות עבודה מקצועיות ספציפיות, בניגוד להתבססות על מודלי-על כלליים. הדבר ניכר בדוגמאות כמו טרמינל Warp שמאפשר לנהל את Claude Code במקביל לסוכנים אחרים למפתחים [3], ובהשקת מודלים מתמחים שעוקפים אנשי מקצוע וכלים כלליים - כמו GPT-5.4 לקלינאים [9] ו-OpenAI Privacy Filter שנועד לרוץ מקומית כדי לשמור על פרטיות נתונים (PII) [4]. מגמה זו מעידה שהתעשייה מספקת כעת כלים שניתן להטמיע בבטחה וביעילות ב-workflows רגישים כמו פסיכיאטריה ופיתוח אלגוריתמים קליניים.

❓ שאלה לצלילה עמוקה

כיצד ניתן למנף את מודל ה-OpenAI Privacy Filter הלוקאלי יחד עם שרת MCP ייעודי, כדי לאפשר ל-Claude Code לגשת בצורה אוטונומית מהטרמינל (בסביבת Warp) לרשומות פסיכיאטריות של מטופלים, ולחלץ מהן תובנות מחקריות ללא זליגה של מידע רגיש לענן?

NVIDIA's New AI Broke My Brain | Two Minute Papers הסרטון מציג את "Sonic", מערכת שליטה רובוטית מולטי-מודאלית חדשה של NVIDIA, המתרגמת קלט אנושי מגוון (וידאו, טקסט, קול או מוזיקה) לתנועות רובוטיות מורכבות במרחב התלת-ממדי [1, 2]. המערכת אומנה על 100 מיליון פריימים של תנועה אנושית ללא צורך בתיוג פעולות ידני, תוך שימוש בארכיטקטורת אנקודר-דיקודר הממירה את הקלט למרחב לטנטי של "אסימונים אוניברסליים" (universal tokens) שמתורגמים לפקודות מוטוריות [3, 4]. תובנה טכנית מרכזית היא יישום של "מודל קפיץ למסלול השורש" (root trajectory spring model), המשמש כבלם פיזיקלי-מתמטי שמרסן פקודות פתאומיות כדי למנוע נזק לרובוט ותנודות אינסופיות [5, 6]. תובנה משמעותית נוספת היא יעילות המודל: הרשת כולה מכילה 42 מיליון פרמטרים בלבד, משקל כה זעיר שמאפשר לה לרוץ באופן מקומי לחלוטין על סמארטפון [3, 7]. עבור פסיכיאטר ילדים ונוער העוסק ב-LLM fundamentals ומפתח סוכנים אופליין, האופן שבו מודל זעיר מצליח לקודד התנהגות אנושית ספציפית ורוויית הבעה (כמו הליכה "שמחה", "חשאית" או "פצועה") לתוך טוקנים מופשטים, רלוונטי במיוחד לפיתוח Agents מולטי-מודאליים שמסוגלים לפרש ניואנסים התנהגותיים בסביבה קלינית מבלי להישען על מודלי עננים כבדים [2, 4, 8].

המגמה הבולטת העולה מהחומר היא המעבר ממודלי-ענק מונוליטיים לסביבות פעולה מבוססות מודלי-קצה (Edge models) זעירים, מולטי-מודאליים ויעילים להחריד, המסוגלים לפענח ולהפיק התנהגות אנושית ניואנסית. דוגמה מובהקת לכך היא רשת ה-42 מיליון פרמטרים של NVIDIA, שמצליחה לזקק קלטים מורכבים במיוחד (הוראות קוליות, וידאו של בני אדם, טקסט) לכדי תנועה פיזית בעלת אופי רגשי [2, 3]. מגמה זו, של יצירת מרחב לטנטי אוניברסלי שדוחס כמות אדירה של מידע לא-מובנה למערכת שיכולה לרוץ על מכשיר נייד [4, 7], משקפת עתיד בו סוכנים אוטונומיים אינם דורשים חוות שרתים, אלא פועלים מקומית בצורה מהירה ומאובטחת לחלוטין.

❓ שאלה לצלילה עמוקה

כיצד ניתן לשאוב השראה מארכיטקטורת ה-"Universal Tokens" (אשר דוחסת תנועות פיזיות והבעות רגשיות אנושיות למרחב לטנטי יעיל) כדי לתכנן סוכני AI מקומיים דרך MCP, שיוכלו לעבד באופן אוטומטי ופרטי התנהגויות מולטי-מודאליות לא-מובנות של ילדים בקליניקה — כגון שפת גוף, אי-שקט מוטורי או טון דיבור — לכדי תובנות קליניות או מחקריות מובנות?

GPT 5.5 הגיע — והסוכנים נכנסים לצ׳אט | ערוץ: עידן והבינה הסרטון סוקר את השקת מודל GPT 5.5, בדגש על יכולות קידוד, דיבוג מתקדם ושימוש עצמאי במחשב דרך טרמינל [1, 2]. כמו כן, מודגמות בו התפתחויות רוחביות בעולם סוכני ה-AI, כולל סוכני סביבת עבודה המשתלבים ב-Slack ויכולות זיכרון חדשות בפלטפורמת הסוכנים של קלוד [3, 4]. תובנה מרכזית אחת שעולה היא ש-OpenAI פיתחה גרסה חינמית וייעודית של ChatGPT עבור רופאים וקלינאים בארה"ב, אשר נשענת על בנצ'מרק רפואי חדש להבטחת דיוק [5]. תובנה נוספת נוגעת להכרזה של גוגל על מנוע סוכני חיפוש אוטונומיים תוך הוספת תמיכה בפרוטוקול MCP (Model Context Protocol), צעד משמעותי שמרחיב את יכולות החיבור למידע מקומי עבור מפתחים [6, 7]. בנוסף, מודגמת היכולת של קלוד לייצר "ארטיפקטים" ויזואליים חיים מתוך שירותים חיצוניים כמו תיבת המייל [8, 9]. עבור פסיכיאטר ילדים ונוער העובד מהטרמינל ונעזר ב-MCP, חומר זה הוא קריטי; הוא ממחיש כיצד ניתן למנף סוכנים בעלי זיכרון לניתוח טקסטואלי רציף של תיקים קליניים, ולשלב כלים בעלי יכולות הנמקה מתקדמות שהותאמו ספציפית לעולם הרפואי והמחקרי [4, 5, 7].

קלוד הפך להיות עורך הווידאו שלי! | ערוץ: בינה מלאכותית עם יובל אבידני YUV AI בסרטון זה מוצגת הדגמה מעשית ומרשימה של שימוש ב-Claude Code להפעלת סוכנים אוטונומיים מורכבים [10, 11]. המציג מראה כיצד באמצעות הרצת סקריפט מבוסס קוד-פתוח משורת הפקודה (Terminal), קלוד מסוגל לנתח קובץ וידאו, להסיר שתיקות, ולהוסיף לו מוזיקה ואנימציות ללא כל התערבות ידנית [10, 11]. פרט משמעותי שעולה מההדגמה הוא אופן החיבור האלגנטי של Claude Code ישירות לממשקי API חיצוניים (כדוגמת ElevenLabs), המופעלים באופן אוטומטי על בסיס פרומפט מובנה [11]. עבור איש מקצוע שחוקר LLM fundamentals ומשלב אוטומציות בסביבת macOS, הסרטון מהווה הדרכה פרקטית יקרת-ערך לבניית תהליכי AI עמוקים ואוטונומיים לחלוטין [10, 11]. בפרספקטיבה מחקרית-קלינית, ניתן להשליך מארכיטקטורה זו על פיתוח סקריפטים לוקאליים שרצים ברקע על המק, מנתחים תמלולים של ראיונות או תצפיות וידאו של בני נוער, מפעילים מודלים חיצוניים לסיווג, ומפיקים דוחות קליניים מובנים בשורת הפקודה [10, 11].

👀 Tier B — שווה סריקה

הופתעתי מהתוצאה — בדיקה בלייב למודל התמונות החדש של צ'אטי | ערוץ: עידן והבינה הסרטון מתמקד בהשוואה ישירה בין יכולות יצירת התמונות של המודל החדש מבית OpenAI (GPT Image 2) לבין ג'מיני של גוגל [12, 13]. הערך המרכזי כאן הוא הדגמת יכולתם המשופרת של המודלים להפיק אינפוגרפיקות עשירות ומפורטות, אשר כוללות שילוב ועימוד של טקסט קריא ישירות בתוך התמונה, כמו למשל תרשימים ביולוגיים הסברתיים על בעלי חיים [14-16]. עבור פסיכיאטר ילדים ונוער, אף שזהו כלי גרפי במהותו, היכולת לייצר חומרי הסברה ויזואליים, מדויקים ונגישים (כמו תרשימי הרגעה או כרטיסיות מידע למטופלים צעירים) יכולה לשמש ככלי עזר פסיכו-חינוכי מצוין בקליניקה [14, 17].

📈 מגמה מרכזית

המגמה הבולטת שעולה מהסרטונים השבוע היא המעבר החד מצ'אטבוטים שממתינים להוראות, לדור חדש של סוכני AI אקטיביים ועצמאיים (Agents), אשר פועלים מתוך סביבות העבודה הטבעיות של המשתמש [3, 4, 11]. ראינו לדוגמה כיצד Claude Code פועל כאוטומציה עצמאית דרך הטרמינל לניהול ממשקי API חיצוניים, לצד פיתוח יכולות זיכרון ארוך-טווח של סוכנים שלומדים מסשן לסשן [4, 11]. במקביל, השילוב של פרוטוקולי קישוריות כמו MCP במערכות הפיתוח של גוגל, והחדרת סוכני עבודה שקוראים הקשר בתוך פלטפורמות תקשורת ארגוניות כ-Slack (מצד OpenAI), מעידים על בשלות משמעותית ביכולות ה-Multi-step reasoning להוצאה לפועל של משימות מורכבות [3, 7].
